# Security Policy

## Supported Versions

We release patches for security vulnerabilities in the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.0   | :white_check_mark: |
| < 0.1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please report it as soon as possible. The responsible disclosure of security vulnerabilities helps us ensure the security and privacy of all our users.

To report a vulnerability, please follow these steps:

1. **Email**: Send an email to oldemar.chaves@gmail.com with the details of the vulnerability.
2. **Include**: In your report, include the following:
    - A detailed description of the vulnerability.
    - Steps to reproduce the vulnerability.
    - Any relevant logs, screenshots, or other supporting information.
3. **Expectations**: You will receive a response within 8 hours acknowledging the receipt of your report. We will work with you to understand the issue and provide a fix as soon as possible.

## Our Commitment

We are committed to addressing and fixing security issues promptly. Once a vulnerability is reported, we will:
- Confirm the vulnerability and its impact.
- Develop a fix and perform extensive testing.
- Release a patch and communicate the update to all users.

Thank you for helping us keep our project safe and secure.
